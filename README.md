# Rutracker AI Filter Bot

[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**Умный фильтр для RSS-ленты Rutracker.org с анализом на основе ИИ и доставкой в Telegram.**

Бот автоматически забирает новые раздачи из публичной RSS-ленты Rutracker, парсит страницы для получения детальной информации (размер, сиды, личи, полное описание), анализирует контент с помощью ИИ (OpenAI или локальная модель), оценивает релевантность и отправляет вам в Telegram только те записи, которые действительно могут быть интересны.

---

## 📌 Основные возможности

- **Автоматический парсинг RSS** – загрузка новых записей по мере публикации.
- **Парсинг страниц раздач** – извлечение размера, количества сидов/личей, категории и полного описания.
- **ИИ-анализ** – оценка полезности (0–100), определение категории, теги, краткое резюме и рекомендация (скачать / возможно / пропустить).
- **Поддержка нескольких ИИ-провайдеров** – OpenAI API, локальные модели (через Ollama), а также встроенные эвристики (без ИИ).
- **Гибкая фильтрация** – отправляются только записи с рейтингом выше заданного порога.
- **Ежедневный дайджест** – вы получаете подборку лучших раздач за сутки прямо в Telegram.
- **Хранение истории** – база данных SQLite сохраняет все обработанные записи, исключая дубли и позволяя отслеживать статус.
- **Модульная архитектура** – каждый этап (RSS, парсинг, ИИ, отправка) работает независимо, что упрощает отладку и расширение.

---

## 🏗 Архитектура

Система построена как **конвейер обработки** (pipeline) из четырёх основных этапов, обменивающихся данными через общую базу данных SQLite.

```
RSS Лента → [RSS Parser] → БД → [Page Parser] → БД → [AI Analyzer] → БД → [Telegram Sender] → Telegram
```

- **RSS Parser** – загружает ленту, сохраняет новые записи с базовыми полями.
- **Page Parser** – для каждой необработанной записи парсит страницу раздачи, извлекает детали.
- **AI Analyzer** – анализирует записи с помощью ИИ или эвристик, проставляет рейтинг и рекомендации.
- **Telegram Sender** – отбирает записи с рейтингом выше порога и отправляет красиво оформленные сообщения.

Каждый этап работает с базой данных через статусные флаги (`is_page_parsed`, `is_ai_analyzed`, `is_sent`), что позволяет запускать их независимо и перезапускать при сбоях.

## 🚀 Установка и запуск

### Требования
- Python 3.8 или выше
- pip и virtualenv (рекомендуется)
- Опционально: API-ключ OpenAI (если используете OpenAI) или локально установленная Ollama (для локальной модели)

### Шаг 1. Клонирование репозитория
```bash
git clone https://github.com/yourusername/rutracker-ai-bot.git
cd rutracker-ai-bot
```

### Шаг 2. Создание виртуального окружения и установка зависимостей
```bash
python -m venv venv
source venv/bin/activate   # Linux/Mac
# или venv\Scripts\activate   # Windows
pip install -r requirements.txt
```

### Шаг 3. Настройка переменных окружения
Создайте файл `.env` в корне проекта или задайте переменные в системе:

```env
# Обязательные
TELEGRAM_TOKEN=your_bot_token_here
CHAT_ID=your_telegram_chat_id_here

# Опционально: для OpenAI
OPENAI_API_KEY=sk-...

# Опционально: для локальной модели (если не используется OpenAI)
USE_LOCAL_AI=true
LOCAL_AI_URL=http://localhost:11434/api/generate
LOCAL_AI_MODEL=llama2
```

Все настройки также можно изменить в `config.py`.

### Шаг 4. Запуск
**Однократный прогон (для теста):**
```bash
python main.py --once
```

**Запуск в бесконечном цикле с интервалом 60 минут:**
```bash
python main.py
```

**Тестирование отдельных этапов:**
```bash
python test_stages.py --rss        # проверить RSS-парсер
python test_stages.py --page       # проверить парсер страниц
python test_stages.py --ai         # проверить ИИ-анализ
python test_stages.py --telegram   # отправить тестовое сообщение
```

## ⚙️ Конфигурация

Все настройки находятся в файле `config.py`. Основные параметры:

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `TELEGRAM_TOKEN` | Токен бота (из @BotFather) | – |
| `CHAT_ID` | ID чата для отправки (из @userinfobot) | – |
| `RSS_URL` | Адрес RSS-ленты | `http://feed.rutracker.cc/atom/f/0.atom` |
| `RSS_MAX_ENTRIES` | Сколько записей загружать за раз | 30 |
| `DB_PATH` | Путь к файлу базы данных | `rutracker.db` |
| `PARSING_DELAY` | Задержка между запросами к сайту (сек) | 0.5 |
| `MIN_RELEVANCE_SCORE` | Минимальный рейтинг для отправки | 50 |
| `MAX_ENTRIES_PER_DAY` | Максимум записей в дайджесте | 20 |
| `OPENAI_API_KEY` | Ключ OpenAI (если используется) | – |
| `USE_LOCAL_AI` | Использовать локальную модель вместо OpenAI | `False` |
| `LOCAL_AI_URL` | URL локальной LLM (например, Ollama) | `http://localhost:11434/api/generate` |
| `LOCAL_AI_MODEL` | Имя модели в локальной LLM | `llama2` |
